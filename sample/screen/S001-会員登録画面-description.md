# S001 会員登録画面

> **出典**: `docs/screen-list.md#S001`, `docs/app-list.md#APP-01`, `docs/service-list.md#SVC-01`, `docs/data-model.md#Member`  
> **作成日**: 2026-03-31

---

## 1. 概要

* **所属アプリケーション**: APP-01（会員・同意管理）
* **目的**: 新規会員がメールアドレス/電話番号/プロフィール情報を入力し、目的別同意を行う。登録完了後に会員 ID を発行してポータルへ誘導する。
* **想定ユーザー**: 未登録の新規顧客
* **前提**: ID基盤（SSO/ソーシャルログイン）連携を想定。SSO ありの場合はメール/パスワード入力をスキップする（TBD: ID基盤仕様、ブロッカー#3）。
* **対応 UC**: UC-01

## 3. ユーザーフロー / 状態

### 主要フロー

1. ユーザーが登録ページにアクセス
2. STEP 1: 基本情報を入力 → バリデーション通過後「次へ」
3. STEP 2: 目的別同意を選択 → 「登録する」を押下
4. API: `POST /api/members` + `POST /api/consents`
5. STEP 3: 登録完了画面を表示 → 「ポータルへ」でS003へ遷移

### SSO ルート（TBD）

1. SSO ボタンを押下 → 外部 ID 基盤へリダイレクト
2. 認証成功後コールバック → STEP 2（同意）のみ実施
3. 完了 → S003へ

### 状態一覧

| 状態 | 説明 |
|---|---|
| 初期 | STEP 1 表示、フォーム空 |
| 入力中 | バリデーション実施中 |
| 送信中 | API 呼び出し中、ボタン無効化・スピナー表示 |
| 登録完了 | STEP 3 完了メッセージ表示 |
| エラー | バリデーション/API エラー表示 |

## 4. 入出力・データ

### 入力項目

| フィールド | 型 | 必須 | 制約 |
|---|---|---|---|
| email | string | ✓ | RFC 5321 形式, 最大 320文字, UNIQUE |
| password | string | ✓ | 8文字以上, 英大小数記号を各1以上 |
| passwordConfirm | string | ✓ | password と一致 |
| name | string | ✓ | 1〜200文字 |
| phone | string | - | E.164形式 (例: +81901234567) |
| consent.marketing | boolean | - | デフォルト: false |
| consent.aiUsage | boolean | - | デフォルト: false |
| consent.thirdParty | boolean | - | デフォルト: false |

### 出力データ

* 登録後: `memberId`（UUID）を発行 → セッション/トークンに格納
* 出典: `docs/data-model.md#Member`, `docs/data-model.md#ConsentRecord`

### API 連携

| 操作 | メソッド | エンドポイント | 備考 |
|---|---|---|---|
| 会員登録 | POST | `/api/members` | SVC-01が処理 |
| 同意登録 | POST | `/api/members/{memberId}/consents` | 目的別3件 |
| SSO コールバック | GET | `/api/auth/callback` | TBD（ID基盤仕様による） |

## 5. バリデーション & エラーメッセージ

| ルール | エラーメッセージ |
|---|---|
| email 未入力 | 「メールアドレスを入力してください」 |
| email 形式不正 | 「正しいメールアドレス形式で入力してください」 |
| email 重複 | 「このメールアドレスはすでに登録されています」 |
| password 長さ不足 | 「パスワードは8文字以上で入力してください」 |
| password 複雑性不足 | 「パスワードには英大文字・英小文字・数字・記号をそれぞれ1文字以上含めてください」 |
| passwordConfirm 不一致 | 「パスワードが一致しません」 |
| name 未入力 | 「氏名を入力してください」 |
| phone 形式不正 | 「電話番号の形式が正しくありません（例: 090-1234-5678）」 |
| API 通信エラー | 「登録処理中にエラーが発生しました。しばらく後に再試行してください」 |

## 6. A11y / i18n

* **キーボード操作**: Tab でフォーカス移動、Enter でフォーム送信（ボタン上）
* **フォーカス順**: ロゴ → ステッパー → email → password → passwordConfirm → name → phone → [次へ]（STEP1）→ consent×3 → PrivacyPolicyLink → [登録する]（STEP2）
* **aria**: 各 input に `aria-required`, `aria-describedby` でエラーメッセージ紐付け。ProgressStepper に `role="progressbar"` + `aria-valuenow`。
* **読み上げ**: エラー発生時に `aria-live="polite"` で画面読み上げに通知
* **色以外の表現**: エラー時は赤色＋エラーアイコン（⚠）＋テキストで表現（色だけに依存しない）
* **言語**: 日本語 UI（i18n 対応: TBD）

## 7. セキュリティ / プライバシー

* **取り扱いデータ分類**: PII（氏名・メール・電話番号）, 同意記録
* **サニタイズ**: 入力値は XSS サニタイズ（フロントエンド: エスケープ、バックエンド: HTML エスケープ）。SQL インジェクション対策はパラメータ化クエリ。
* **パスワード保護**: クライアント側でパスワードをログ出力禁止。送信は HTTPS のみ。サーバー側は bcrypt/Argon2 でハッシュ保存（SVC-01側）。
* **保存範囲**: セッション中のみ（パスワードのブラウザ保存はデフォルト無効 `autocomplete="new-password"` で誘導）
* **同意記録**: 目的別に SVC-01 が `ConsentRecord` に保存（Append-only）。同意変更時は S002 へ誘導。
* **TBD**: 同意要件の法域・目的別オプトイン粒度（ブロッカー#1）

## 8. 非機能要件

* **パフォーマンス**: 登録 API のレスポンスタイム p95 < 1,000ms（TBD: SVC-01 SLA確定後に更新）
* **可用性**: APP-01 の可用性 99.9%（`docs/app-list.md#APP-01`）
* **ログ**: 登録成功/失敗を `AuditLog` に記録（eventType: `MEMBER_REGISTERED`）。PII はマスキング（TBD: マスキング要件、ブロッカー#11）

## 9. 受け入れ基準（テスト可能）

| # | Given | When | Then |
|---|---|---|---|
| AC-001 | 未登録ユーザーが登録フォームを開いた | 正しい形式でメール・パスワード・氏名を入力し「登録する」を押す | 会員 ID が発行され、ポータル（S003）へ遷移する |
| AC-002 | 登録フォームに無効なメールアドレスを入力した | 「次へ」を押す | 「正しいメールアドレス形式で入力してください」エラーが表示される |
| AC-003 | すでに登録済みのメールアドレスを入力した | 「登録する」を押す | 「このメールアドレスはすでに登録されています」エラーが表示される |
| AC-004 | パスワードと確認パスワードが異なる値を入力した | 「次へ」を押す | 「パスワードが一致しません」エラーが表示される |
| AC-005 | STEP2 で同意チェックを1つも付けずに登録した | 「登録する」を押す | 登録は成功し、同意状態はすべて `REVOKED` で保存される |
| AC-006 | STEP2 でマーケティング同意にチェックを付けた | 「登録する」を押す | ConsentRecord に purpose=MARKETING, status=GRANTED で保存される |
