# S004 ポイント残高・明細照会画面

> **出典**: `docs/screen-list.md#S004`, `docs/app-list.md#APP-04`, `docs/service-list.md#SVC-02`, `docs/data-model.md#LoyaltyAccount`, `docs/data-model.md#PointLedgerEntry`  
> **作成日**: 2026-03-31

---

## 1. 概要

* **所属アプリケーション**: APP-04（顧客エンゲージメントポータル）
* **目的**: 現在のポイント残高・ランク・付与/消込の取引履歴を一覧表示する。会員が自身のポイント動向を把握するための主要参照画面。
* **想定ユーザー**: 認証済み会員（顧客）
* **前提**: S003 の「残高タブ」または直接アクセス。SVC-02（ポイント台帳サービス）を参照。
* **対応 UC**: UC-03

---

## 2. 画面構成

### レイアウト概要

```
┌───────────────────────────────────────────┐
│  ナビゲーションバー（← ポータルへ戻る）           │
├───────────────────────────────────────────┤
│  残高サマリーカード:                            │
│    有効ポイント: [XXXX pt]                       │
│    ランク: [GOLD]                               │
│    最終更新: [YYYY-MM-DD HH:mm]                 │
├───────────────────────────────────────────┤
│  フィルター: [期間 ▼] [種別 ▼: 全て/付与/消込/失効] │
├───────────────────────────────────────────┤
│  取引履歴一覧（ページネーション付き）               │
│  ┌──────────────────────────────────────┐ │
│  │ 日付    │ 内容         │ 変動  │ 残高  │ │
│  │ 03/15  │ 購買付与      │ +100 │ 3500 │ │
│  │ 03/14  │ 特典交換（消込）│ -500 │ 3400 │ │
│  │ ...    │ ...          │ ...  │ ...  │ │
│  └──────────────────────────────────────┘ │
│  [< 前ページ] [1][2][3] [次ページ >]           │
└───────────────────────────────────────────┘
```

### コンポーネント一覧

| コンポーネント | 種別 | 備考 |
|---|---|---|
| BalanceSummaryCard | 表示 | 残高・ランク・更新時刻 |
| PeriodFilter | 操作 | 期間選択（1ヶ月/3ヶ月/6ヶ月/カスタム）|
| TypeFilter | 操作 | 種別フィルター（全て/AWARD/REDEEM/EXPIRE/ADJUSTMENT）|
| LedgerTable | 表示 | 取引履歴テーブル |
| Pagination | 操作 | ページネーション（20件/ページ）|
| LoadingSkeleton | 表示 | データ取得中 |
| EmptyState | 表示 | 履歴ゼロ時のメッセージ |

## 3. ユーザーフロー / 状態

### 主要フロー

1. 画面アクセス時に残高サマリーと明細（最新20件）を取得
2. フィルター操作 → 条件に合わせて API 再取得
3. ページネーション操作 → 続きのデータ取得

### 状態一覧

| 状態 | 説明 |
|---|---|
| 読み込み中 | スケルトン表示 |
| 表示中 | 残高・明細一覧表示 |
| 空（履歴なし） | 「取引履歴はまだありません」 |
| フィルター結果ゼロ | 「指定した条件の履歴はありません」 |
| エラー | 「データを取得できませんでした」＋再試行ボタン |

## 6. A11y / i18n

* **キーボード操作**: Tab でフォーカス。テーブルは `role="grid"` で矢印キーナビゲーション。
* **フォーカス順**: 戻るリンク → BalanceSummaryCard → PeriodFilter → TypeFilter → LedgerTable → Pagination
* **aria**: LedgerTable に適切な `<thead>`/`<th scope="col">`. 変動の正負は `aria-label="+100ポイント付与"` で読み上げ。
* **色以外の表現**: 付与は緑＋「+」アイコン、消込は赤＋「-」アイコン＋テキストで区別。
* **言語**: 日本語（金額・日付のロケールフォーマット: ja-JP）

---

## 7. セキュリティ / プライバシー

* **取り扱いデータ分類**: ポイント台帳データ（非 PII、ただし memberId と紐付くため準センシティブ）
* **認証**: Bearer トークン必須。他会員データへのアクセス禁止（バックエンドで `memberId` をトークンから取得）
* **ログ**: 台帳照会は監査ログ不要（顧客自身のデータ参照のため）。管理者アクセス（S015）は別途ログ記録。

---

## 8. 非機能要件

* **パフォーマンス**: 台帳明細取得 p95 < 800ms（SVC-02、`docs/service-list.md#SVC-02`）
* **更新時刻表示**: 残高の最終更新時刻を表示（取得元のサーバー時刻）
* **ページサイズ**: デフォルト20件。最大100件（TBD: サービス仕様確定後）

## 9. 受け入れ基準（テスト可能）

| # | Given | When | Then |
|---|---|---|---|
| AC-001 | ログイン済み会員がポイント残高・明細画面を開いた | ページ表示が完了した | 現在の有効ポイント・ランク・最新の取引履歴が表示される |
| AC-002 | 取引履歴が表示されている | 期間フィルターを「先月」に変更した | 先月の取引のみ表示され、件数が更新される |
| AC-003 | 取引履歴が表示されている | 種別フィルターを「付与」に変更した | type=AWARD の取引のみ表示される |
| AC-004 | 取引履歴が20件以上ある | 「次のページ」を押した | 次ページのデータが表示される |
| AC-005 | 購買後にポイントが付与された | ページを再読み込みした | 付与されたポイントが明細に表示されている |
