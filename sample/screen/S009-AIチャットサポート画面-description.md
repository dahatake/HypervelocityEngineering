# S009 AIチャットサポート画面

> **出典**: `docs/screen-list.md#S009`, `docs/app-list.md#APP-08`, `docs/service-list.md#SVC-10`  
> **作成日**: 2026-03-31

---

## 1. 概要

* **所属アプリケーション**: APP-08（AI CSサポート）
* **目的**: チャット形式で AI に問い合わせを行い、必要に応じて CS 担当へ引き継ぐ。個人情報マスキング後に引継ぎ。有人エスカレーションボタンを含む。
* **想定ユーザー**: 認証済み会員（顧客）
* **前提**: S003 の「サポート」リンクから遷移。SVC-10（AI CSサポートサービス）と接続。
* **対応 UC**: UC-06

---

## 3. ユーザーフロー / 状態

### AI 対応フロー

1. 画面表示 → WebSocket / SSE で SVC-10 に接続
2. ユーザーがメッセージ送信
3. AI が応答生成 → ストリーミング表示
4. 解決 → セッション終了

### エスカレーションフロー

1. 「担当者に繋いでほしい」押下
2. 個人情報マスキング処理（SVC-10 側）
3. CS担当に引き継ぎ情報（会話要約）送信
4. ステータス「担当者が対応します」に変更（S028: CS担当側へ通知）

### 状態一覧

| 状態 | 説明 |
|---|---|
| 接続中 | WebSocket 接続待機 |
| AI 対応中 | AI 応答可能 |
| AI 生成中 | TypingIndicator 表示 |
| エスカレーション申請中 | 「引き継ぎ中」表示 |
| CS担当対応中 | 「担当者が対応します」表示 |
| セッション終了 | 満足度評価 UI |
| 接続エラー | 「接続に問題が発生しました。再接続してください」|

## 4. 入出力・データ

### 入力

| フィールド | 制約 |
|---|---|
| message | 最大1,000文字 |

### 表示データ

| データ | 出典 |
|---|---|
| チャット履歴 | SVC-10（セッション内メモリ）|
| エスカレーション状態 | SVC-10 のセッションステータス |

### API 連携

| 操作 | プロトコル | エンドポイント | 備考 |
|---|---|---|---|
| チャット接続 | WebSocket / SSE | `/ws/chat` または `/api/chat/stream` | SVC-10 |
| メッセージ送信 | POST | `/api/chat/messages` | または WebSocket 経由 |
| エスカレーション申請 | POST | `/api/chat/escalate` | SVC-10。PII マスキング後に CS 通知 |

## 5. バリデーション & エラーメッセージ

| ルール | エラーメッセージ |
|---|---|

## 6. A11y / i18n

* **キーボード操作**: Enter でメッセージ送信（Shift+Enter で改行）。Tab でエスカレーションボタンへ移動。
* **フォーカス順**: ChatInput → 送信ボタン → EscalationButton
* **aria**: ChartWindow に `role="log"`, `aria-live="polite"`. 新着メッセージは自動読み上げ。TypingIndicator に `aria-label="AIが応答を作成中"`.
* **色以外の表現**: AI/ユーザーのメッセージ区別はバブルの位置（左/右）＋送信者ラベル（AI/あなた）で表現。
* **言語**: 日本語

## 7. セキュリティ / プライバシー

* **取り扱いデータ分類**: 問い合わせ内容（潜在的 PII）、会員 ID
* **PII マスキング**: エスカレーション時、SVC-10 がチャット履歴内の PII（氏名・住所・電話等）を自動マスキングしてからCS担当に引き継ぎ
* **チャットログ保存**: セッションログはナレッジDB向けの改善用として保存可（TBD: 保存期間・同意要件）
* **WebSocket 認証**: 接続時に Bearer トークン検証
* **TBD**: ナレッジDBの所在・更新責任（ブロッカー #5）

## 8. 非機能要件

* **パフォーマンス**: AI 初回応答 p95 < 3,000ms（生成 AI の特性上）
* **接続維持**: WebSocket ハートビート 30秒間隔
* **エスカレーション**: 申請後 CS 担当への通知は < 5秒

## 9. 受け入れ基準（テスト可能）

| # | Given | When | Then |
|---|---|---|---|
| AC-001 | 会員がAIチャット画面を開いた | 表示完了した | チャットウィンドウが表示され、メッセージ入力が可能な状態になる |
| AC-002 | チャット画面でメッセージを入力した | 「送信」を押した | AI の応答がチャットウィンドウに表示される |
| AC-003 | AI との会話中に「担当者に繋いでほしい」を押した | エスカレーション処理が完了した | ステータスが「担当者が対応します」に変わり、CS担当画面（S028）に通知が届く |
| AC-004 | エスカレーション時に会話内に個人情報が含まれていた | 引き継ぎ処理が実行される | CS担当に渡される要約の個人情報はマスキングされている |
| AC-005 | メッセージを空のまま「送信」を押した | バリデーションが実行される | 「メッセージを入力してください」エラーが表示され送信されない |
