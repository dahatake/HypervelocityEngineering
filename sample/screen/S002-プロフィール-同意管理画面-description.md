# S002 プロフィール・同意管理画面

> **出典**: `docs/screen-list.md#S002`, `docs/app-list.md#APP-01`, `docs/service-list.md#SVC-01`, `docs/data-model.md#Member`, `docs/data-model.md#ConsentRecord`  
> **作成日**: 2026-03-31

---

## 1. 概要

* **所属アプリケーション**: APP-01（会員・同意管理）
* **目的**: 既存会員が自己プロフィール（氏名・連絡先）と目的別同意設定（マーケ/AI利用/第三者）を確認・更新する。GDPR 同意撤回フローを含む。
* **想定ユーザー**: 既存会員（認証済み）
* **前提**: ログイン済みセッション必須。`memberId` はセッションから取得。
* **対応 UC**: UC-01（Secondary: 同意変更）

## 3. ユーザーフロー / 状態

### プロフィール更新フロー

1. ページ表示時に `GET /api/members/{memberId}` でプロフィール取得
2. 氏名/電話番号を編集 → 「保存する」押下
3. API: `PATCH /api/members/{memberId}` でプロフィール更新
4. 成功: インラインバナー「プロフィールを更新しました」

### 同意設定更新フロー

1. ページ表示時に `GET /api/members/{memberId}/consents` で現在の同意状態取得
2. トグル操作 → 「同意設定を保存」押下
3. API: `PUT /api/members/{memberId}/consents` で更新
4. 成功: インラインバナー「同意設定を更新しました」

### GDPR 同意撤回フロー

1. 全同意を OFF にして保存
2. データ削除申請リンク（退会申請）から削除フロー

### 状態一覧

| 状態 | 説明 |
|---|---|
| 初期（読み込み中） | スケルトン表示 |
| 表示中 | プロフィール・同意情報表示 |
| 更新中 | API 呼び出し中、ボタン無効化 |
| 更新成功 | インラインバナー表示（3秒後自動消去）|
| 更新失敗 | エラーバナー表示 |

## 6. A11y / i18n

* **キーボード操作**: Tab でフォーカス移動。トグルは Space/Enter で切り替え。
* **フォーカス順**: ナビゲーション → プロフィールフォーム（氏名→電話番号→[保存]）→ 同意トグル×3 → [同意保存] → 退会リンク
* **aria**: ConsentToggle に `role="switch"`, `aria-checked`, `aria-label`。保存成功/失敗の AlertBanner に `aria-live="polite"`。
* **読み上げ**: トグル変更時に「マーケティング利用 オン/オフ」をスクリーンリーダーに通知。
* **色以外の表現**: トグル ON/OFF はアイコン（●/○）＋ラベル「同意する/同意しない」で表現。
* **言語**: 日本語

## 7. セキュリティ / プライバシー

* **取り扱いデータ分類**: PII（氏名・電話番号）, 同意記録（センシティブ）
* **サニタイズ**: XSS 対策（入力エスケープ）、CSRF 対策（SameSite Cookie + CSRF トークン）
* **同意変更の即時反映**: 同意撤回後はマーケ配信等の処理に即時反映（SVC-01 → イベント発行）
* **退会申請**: `status = WITHDRAWN` へ更新。データ保持期間は TBD（ブロッカー#11）。
* **監査ログ**: プロフィール変更・同意変更は `AuditLog` に記録（SVC-16経由）
* **TBD**: メールアドレス変更フロー（メール認証必須か TBD）

## 8. 非機能要件

* **パフォーマンス**: プロフィール取得 p95 < 500ms（TBD: SVC-01 SLA）
* **同意変更即時反映**: 変更後 < 5分以内に全配信制御に反映（推論: リアルタイム反映は技術的難易度高のため5分以内を許容範囲と設定。この回答はCopilot推論をしたものです。）
* **可用性**: APP-01 99.9%（`docs/app-list.md#APP-01`）

## 9. 受け入れ基準（テスト可能）

| # | Given | When | Then |
|---|---|---|---|
| AC-001 | ログイン済み会員がプロフィール画面を開いた | 現在の氏名・電話番号・同意状態が正しく表示される | 各フィールドに DB 保存値が表示されている |
| AC-002 | 氏名を変更して「保存する」を押した | PATCH リクエストが送信される | 「プロフィールを更新しました」バナーが表示され、DB が更新される |
| AC-003 | マーケティング同意トグルを OFF にして「同意設定を保存」を押した | PUT リクエストが送信される | ConsentRecord に purpose=MARKETING, status=REVOKED で Append 保存される |
| AC-004 | 氏名を空にして「保存する」を押した | バリデーションが実行される | 「氏名を入力してください」エラーが表示され API は呼ばれない |
| AC-005 | 未ログイン状態でこのページにアクセスした | 認証チェックが実行される | ログインページへリダイレクトされる |
