# S003 顧客ポータル（ホーム）

> **出典**: `docs/catalog/screen-catalog.md#S003`, `docs/catalog/app-catalog.md#APP-04`, `docs/catalog/service-catalog.md#SVC-05`, `docs/catalog/data-model.md#LoyaltyAccount`, `docs/catalog/data-model.md#Reward`  
> **作成日**: 2026-03-31

---

## 1. 概要

* **所属アプリケーション**: APP-04（顧客エンゲージメントポータル）
* **目的**: ポイント残高・ランク・おすすめ特典・最新オファーをタブ形式で表示するポータルホーム。UC-03 の主要エントリポイント。APP-02/APP-03/APP-06 のデータを集約表示する。
* **想定ユーザー**: 認証済み会員（顧客）
* **前提**: ログイン済みセッション必須。BFF（SVC-05）経由で各バックエンドサービスからデータを集約。
* **対応 UC**: UC-03（Primary）

## 3. ユーザーフロー / 状態

### 主要フロー

1. 認証済みユーザーがポータルにアクセス
2. BFF（SVC-05）が並列で以下を取得:
   - `GET /api/accounts/{memberId}/summary`（APP-02: 残高・ランク）
   - `GET /api/rewards/recommended`（APP-03: おすすめ特典）
   - `GET /api/offers/personalized`（APP-06: パーソナライズオファー）
3. ポータルホーム表示
4. タブ操作でS004/S005/S006へ遷移

### 状態一覧

| 状態 | 説明 |
|---|---|
| 読み込み中 | スケルトン UI 表示（並列 API 実行中）|
| 表示中（正常） | 全データ表示 |
| 部分エラー | 一部 API 失敗時は「データ取得エラー」プレースホルダー表示（他データは正常表示）|
| 全エラー | 「データを取得できませんでした。再読み込みしてください」|

## 4. 入出力・データ

### 表示データ

| データ | 出典 | API |
|---|---|---|
| 会員名 | `Member.name`（APP-01） | `GET /api/members/{memberId}` |
| ポイント残高 | `LoyaltyAccount.balance`（APP-02） | SVC-05 経由 |
| ランク | `LoyaltyAccount.rank`（APP-02） | SVC-05 経由 |
| 次ランクまで必要ポイント | `LoyaltyAccount.balance` vs `RankRule.threshold`（APP-02） | SVC-05 経由 |
| おすすめ特典（最大3件） | `Reward`（APP-03） | SVC-05 経由 |
| パーソナライズオファー件数 | APP-06 推薦スコア | SVC-05 経由 |

### API 連携

| 操作 | メソッド | エンドポイント | 備考 |
|---|---|---|---|
| ポータルサマリー取得 | GET | `/api/portal/summary` | SVC-05 BFF が集約 |

## 6. A11y / i18n

* **キーボード操作**: Tab でフォーカス移動。タブナビゲーションは矢印キーで切り替え。
* **フォーカス順**: ナビバー → WelcomeBanner → PointSummaryCard → TabNavigation → タブコンテンツ → RewardCarousel → BottomNavBar
* **aria**: TabNavigation に `role="tablist"`, 各タブに `role="tab"`, `aria-selected`. タブパネルに `role="tabpanel"`, `aria-labelledby`.
* **読み上げ**: PointSummaryCard の残高は `aria-label="有効ポイント XXXX ポイント、ランク GOLD"` で読み上げ。
* **色以外の表現**: ランクは色＋ランクテキスト（BRONZE/SILVER/GOLD/PLATINUM）の両方で表現。
* **言語**: 日本語

## 7. セキュリティ / プライバシー

* **取り扱いデータ分類**: 会員名（PII）, ポイント残高（非PII）
* **認証**: Bearer トークン（JWT）または Cookie（HttpOnly, Secure, SameSite=Strict）
* **CSRF**: SameSite Cookie + Origin チェック（読み取り専用画面のため GET のみ）
* **データ最小化**: ポータルサマリーには必要最小限のデータのみ返す（BFF で集約）
* **PII ログ除外**: 会員名等の PII を API ログ/エラートレースに含めない

---

## 8. 非機能要件

* **パフォーマンス**: ポータルサマリー取得 p95 < 800ms（SVC-05 BFF が並列集約）
* **キャッシュ**: おすすめ特典は最大5分 TTL でキャッシュ可（SVC-05 判断）
* **更新時刻表示**: 「最終更新: XX:XX」を常時表示（実装例: サーバー時刻）
* **可用性**: APP-04 99.9%（`docs/catalog/app-catalog.md#APP-04`）

## 9. 受け入れ基準（テスト可能）

| # | Given | When | Then |
|---|---|---|---|
| AC-001 | ログイン済み会員がポータルにアクセスした | ページ読み込みが完了する | 会員名・ポイント残高・ランクが正しく表示される |
| AC-002 | ポータルを表示している | 「残高・明細」タブを押す | S004（ポイント残高・明細照会画面）のコンテンツが表示される |
| AC-003 | ポータルを表示している | おすすめ特典の1件をクリックする | S007（特典詳細・交換確認画面）へ遷移する |
| AC-004 | APP-02 のAPIが500エラーを返した | ポータルを表示した | 残高セクションは「データを取得できませんでした」を表示し、他のセクションは正常表示される |
| AC-005 | 未ログイン状態でポータルURLにアクセスした | ページ表示しようとする | ログインページへリダイレクトされる |
